# 📋 Сводка системы экспедиций

> *Полный обзор модуля экспедиций и заказов на оружие для интеграции в основной проект.*

---

## 🎯 Назначение модуля

Модуль экспедиций — это **отдельная подсистема**, которая работает параллельно с основным проектом. Она отвечает за:

1. **Экспедиции** — миссии авантюристов с оружием игрока
2. **Заказы на оружие** — основной заработок кузнеца
3. **Наполнение мира** — NPC, локации, лор

## 📊 Две основные системы

### 1. Экспедиции

**Зачем:** Отправка авантюристов с оружием игрока для получения ресурсов и душ.

**Вход:**
```typescript
{
  adventurerId: 'warrior',    // ID авантюриста
  weaponId: 'player_sword_1', // ID оружия ИЗ ОСНОВНОГО ПРОЕКТА
  location: 'forest',         // Локация
  difficulty: 'normal'        // Сложность
}
```

**Выход:**
```typescript
{
  totalSoul: 150,              // Души для оружия
  totalMaterials: [...],       // Ресурсы в инвентарь
  events: [...],               // Лог событий
  success: true                // Успех/провал
}
```

**Что делает основной проект:**
1. Передаёт ID оружия игрока
2. Получает результат экспедиции
3. Добавляет души к оружию
4. Добавляет ресурсы в инвентарь

---

### 2. Заказы на оружие

**Зачем:** Основной заработок кузнеца.

**Вход (от основного проекта):**
```typescript
{
  availableRecipes: [         // Все рецепты оружия
    { id: 'iron_sword', name: 'Железный меч', weaponType: 'sword', tier: 1, baseValue: 50 },
    // ... все рецепты
  ],
  playerTier: 2,              // Тир игрока
  relations: {                // Отношения с NPC
    grem_elder: 25,
    korto_woodsman: 10
  }
}
```

**Выход (от модуля):**
```typescript
[
  {
    id: 'order_grem_sword_123',
    npcId: 'grem_elder',
    recipeId: 'iron_sword',      // ID рецепта ДЛЯ ОСНОВНОГО ПРОЕКТА
    weaponName: 'Железный меч',
    quantity: 1,
    goldReward: 55,
    relationGain: 5,
    requestText: 'Мне нужно оружие...',
    completionText: 'Хорошо...'
  },
  // ... другие заказы
]
```

**Что делает основной проект:**
1. Передаёт список рецептов и состояние игрока
2. Получает список заказов
3. Отображает заказы в UI
4. Игрок создаёт оружие через систему крафта
5. Доставляет оружие → получает золото и отношения

## 💾 Сохранение состояния

Модуль **не хранит состояние**. Всё сохранение делает основной проект:

```typescript
// Основной проект сохраняет:
interface PlayerState {
  // Экспедиции
  activeExpeditions: ExpeditionState[]
  
  // Заказы
  activeOrders: ActiveWeaponOrder[]
  completedOrders: string[]
  
  // Отношения
  npcRelations: Record<string, number>
  
  // Ресурсы
  inventory: Record<string, number>
  
  // Оружие и души
  weapons: Weapon[]
  weaponSouls: Record<string, number>
}
```
